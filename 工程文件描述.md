# 鸿蒙基础工程文件解析文档
## 1. 项目基础信息
1. 项目来源：OpenHarmony官方applications_app_samples开源仓库
2. 项目名称：ArkTSClock简易时钟
3. 开发模型：Stage模型
4. 开发语言：ArkTS
5. 运行环境：DevEco Studio + OpenHarmony模拟器

## 2. 项目整体目录结构
ArkTSClock/
├── entry/                        # 应用唯一主业务模块，Stage模型下所有页面、逻辑、资源均存放于此
│   └── src/main/                 # 业务主目录，存放应用全部业务代码与静态资源
│       ├── ets/                  # ArkTS业务代码根目录
│       │   └── pages/            # 页面路由目录，存放所有页面组件文件
│       │       └── Index.ets     # 应用唯一首页，时钟核心绘制、时间刷新逻辑全部在此文件实现
│       ├── resources/base/       # 全局静态资源目录，区分多语言、尺寸、样式资源
│       │   ├── element/          # 尺寸、数值常量资源（文字大小、边距、画布尺寸等）
│       │   ├── media/            # 图片、图标静态素材
│       │   └── string/           # 多语言文本字符串（界面提示文字）
│       └── module.json5          # 模块配置文件，Stage模型核心配置，定义页面路由、默认启动页、权限、窗口属性
├── .hvigor/                      # 构建工具缓存目录，hvigor构建框架自动生成
├── .idea/                        # DevEco Studio编辑器配置缓存，存放界面布局、运行配置
├── build-profile.json5           # 工程全局构建配置，定义编译参数、目标设备、API版本、签名配置
├── hvigorfile.ts                 # 构建脚本文件，自定义编译、打包、调试执行逻辑
├── oh-package.json5              # 项目依赖配置文件，管理第三方开源依赖、包名、版本、应用标识
├── oh-package-lock.json5         # 依赖锁定文件，固定所有依赖包版本，保证多设备构建环境一致
├── README.md                     # 英文官方工程说明文档，描述工程功能、使用限制、运行要求
├── README_zh.md                  # 中文官方工程说明文档
└── ohosTest/                     # 单元测试目录，存放应用自动化测试代码

## 3. 核心代码文件功能说明
1. entry/src/main/ets/pages/Index.ets
时钟首页入口，实现实时时分秒动态刷新、页面布局、时间渲染逻辑。
2. module.json5
配置应用默认启动页面，注册页面路由信息。
3. resources资源目录
统一存放文本字符串、尺寸样式等静态资源。
4. oh-package.json5
记录项目版本、第三方依赖包信息。

## 4. 运行说明
使用DevEco Studio内置鸿蒙模拟器，导入该工程即可直接编译运行。
